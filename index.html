Introduction

We are surrounded by technology, it is used in both our daily lives and work lives. With this being said, there are countless modern businesses which uses technology such as IBM, Microsoft, Google, Schools, Sony, and a lot more. While technology comes with certain benefits which allows businesses to efficiently be productive, it has also exposed businesses to certain vulnerabilities which left unchecked, can cause vital damage. This website will teach you security Mechanisms which helps defends businesses using technology against vulnerabilities.

The basics: What is cybersecurity?
Cybersecurity is the way individual people and organisations try to remove the risk of being affected by some kind of cybercrime. Basically, cybersecurity exists to help protect the devices we use (e.g. smartphones, laptops and computers) and the services we access online. It's also about protecting the personal information we store on these devices and online. Since technologies like smartphones and the internet are now a fundamental part of society, it's more important than ever to prevent cyber criminals from taking our accounts, data and devices.

How do I stay secure?
The National Cyber Security Center has determined six "cyber aware" actions you can take to stay secure online.

  1. Use a strong password for your email.
  2. Install the latest software and app updates.
  3. Turn on 2-Step Verification (2SV).
  4. Use a password manager.
  5. Back up your data.
  6. Create strong passwords out of three random words.

Assets

An asset can be many things, a major application, an item of value to stakeholders, it can be tangible (servers that house their data) or non-tangible (the data itself). Regardless of what the asset is, an asset is always something that is considered highly valuable to a business.
In the cases present on this web page, an asset could be user data, something the business values highly and doesn't want others to be able to access it outside of their company.

Assets can change over time and there should be constant reviews of the company to see what their most valuable assets are so they know wehere to focus their security.

    Access Control

    The management of access to network and information systems. Which users can access data or services and those that can are appropriately verified, authenticated and authorised.
    It is imperative that an organisation is clear about who authorisation to interact with network and information systems supporting an essential function in any way or access associated sensitive data.
    Rights granted should be carefully controlled, especially when those rights provide an ability to materially affect the operation of an essential function. Its like a printer room at a school, only teachers and those with passes are allowed to access the printers, but only the IT department are allowed to change the printer functions. This is done so that everything is done as efficiently as possible.
    These rights should also be reviewed periodically and removed when no longer required. E.g. once a student has used the printers using their pass, they give the pass back to the teacher and therefore no longer have access to the printer room. From a business standpoint, this might be when an individual leaves the organisation.

    Users, devices, and systems should all be verified and reviewed so that nothing unwanted has attached itself to anything.

    Information Security

    Confidentiality

    The ability to protect data so that unauthorised parties cannot view the data. Confidentiality is like a secret a friend tells you. You now posses confidential information about them. It is your duty to protect the confidentiality so that no one else knows.
    In business, this data typically consists of:

    • Personal data: national insurance numbers, full names, phone numbers, addresses, email addresses, credit card numbers, etc.
    • Trade secrets: customer and supplier lists, source codes, processes, inventions, etc.
    • Other resitricted business data: unpublished financial information.

    Confidentiality is to ensure that private data, such as those listed above, are protected against unauthorised access, disclosure without permission, and theft.
    The biggest enemy to confidentiality are data breaches, which are extremely dangerous for businesses, causing significant financial losses and reputational damage. Like letting the secret about your friend slip, it may ruin their reputation in the school (not nice).

    Business imploy methods to ensure data confidentiality such as restricting data.
    Only certain individuals in the organisation have access to this high value private data. This reduces the probability of a random slip up from any number of the organisations employees.
    Other ways include the encryption of data (so when transferring it cannot be understood by anyone without a key), implementing a confidentiality policy/devlopment and implementation of a cybersecurity program (ensures employees understand how to handle confidential data so it doesn't end up in the wrong hands), physical security measures (alarm systems, surveillance cameras etc.) and non-disclosure agreements (a contract that states the individuals who recieve confidential data, such as stakeholders, will not disclose the information).

    Data Security

    Encryption

    The NCSC defines encryption as: “Protection of information by making it unreadable by everyone except those with the key to decrypt it.”
    More specifically, an encryption algorithm uses a key - which is usually a random string of characters - to encode data or a message, making it unreadable until it is decrypted again.
    Encryption DOESN’T stop someone from intercepting a message, it just prevents them from understanding it.
    There are a lot of different times encryption might be used:

    • When sending sensitive information across the internet. Any website that says ‘https’ and all modern iOS and Android apps all use encryption to send data.
    • When storing sensitive information on a mobile device such as a laptop. Since it could potentially be easily stolen, encrypting this data and locking it behind a username and password makes it unreadable to any potential thief.
    • TV transmissions may use encryption to prevent users who do not have a subscription from watching certain TV shows.

    Computer Security

    Physical Security

    The NCSC defines physical security as: “Protecting the places where people work, and the locations of systems, services and networks.”
    This is a broad security mechanism that aims to protect from a range of threats such as fire, flooding, theft, vandalism and terrorism. Good physical security relies on three main things:

    • Access Control
      Described more in-depth       here. But specifically for physical security access control may also include infrastructure that defends against environmental threats and natural disasters e.g. walls/barriers to defend against earthquakes, floods, etc.
    • Surveillance
      Surveillance technology such as CCTV cameras can be helpful in not only providing evidence and capturing criminal behaviour but also in stopping it preemptively. (People are far less likely to break into a building if they see obvious cameras).
    • Testing
      Testing is an important part of physical security too. In order to know how well a company can respond to or stop a threat they have to do tests. An example of this could be something as simple as a fire drill, or as complex as specific tests on a company’s policies and plans in case of some sort of disaster.

    Authentication

    The process or action of verifying the identity of a user or process.

    Why do we authenticate? Its important to make sure someone or something is who they say they are before allowing them to do what they will. In the same way its good to check the person at the door is who they claim to be by looking at them through the peephole. You don't want anyone to be able to access your things incase they steal or damage them in any way. In the computing world, we call this User Authentication. It is the main method for protecting access to devices and the data stored on smartphones, tablets, laptops, and desktops. It also protects against unwanted changes (e.g. administrator settings on windows).

    Businesses will use something called CDS (cross domain solutions) to authenticate all users, equipment, and internal component connection sessions before use of any systems.
    User authentication is important so our data isn't stolen or tampered with.
    Equipment authentication is important so that any virus carrying or tampered with equipment cannot access our data.
    Components within the CDS are authenticated to each other, ensuring the flow of data and authorisations within the system are as intended.

    Various authentication methods can be used depending on what service is being accessed and by what, each method having its own strengths and weaknesses.

    Its important for businesses to consider both the security and usability of their devices and services. If they implement the most secure system ever created, but it takes users too long to access, then the security mechanism is pointless, as it cannot be easily accessed by trusted users.

    Attackers look to target weaknesses in authentication systems. Many common attacks look for simple ways to guess or steal user device credentials.
    With these credentials, attackers can impersonate valid users and devices, gaining access to valuables (e.g. data), and the ability to connect remotely to enterprise services, using this foothold to penetrate further into corporate networks.
    Given these potential consequences, it should be clear that implementing effecting authentication is essential for organisations wishing to protect against account, device or netowrk compromise.

    Network Security

    Firewall

    When it comes to businesses implementing an effective cyber security strategy. Having a firewall can make a huge contribution to this. A firewall can be thought of as a protective force field, keeping a businesses’ internal network infrastructure safe from any external attacks which may cause harm to that network infrastructure. How firewalls normally works is they will monitor ingoing and outgoing network traffic based typically on pre-defined security rules. The main goals for using a firewall is to prevent any data breaches, unauthorized access, malware infections and any other form of cyber security attacks.
    There are a lot of advantages for businesses using a firewall:

    • A firewall acts as an extra layer to network security through the use of inspecting and filtering network traffic in order to defend against cyber security attacks. This works by detecting and blocking any suspicious activity, preventing any unauthorised access.
    • Businesses are also able to use firewall for even multiple purposes as well. Most modern firewalls nowadays has built in advanced threat detection systems such as an intrusion detection system and malware detection systems as. Overall, businesses are able to use firewalls for multiple purposes which can go beyond the scope of just network trafficking.
    • Businesses are able to use firewalls to determine access policies. This allows users to be restricted to have access on only the resources they need within the network. This helps to prevent data leaks and any unauthorized access to sensitive information.
    • It is thanks to firewalls being able to track network traffic, businesses are able to have better insight on their network patterns which allows them to identify any vulnerabilities and security gaps. This can even help businesses to make decisions regarding how they can further strengthen their own cyber security infrastructure.
